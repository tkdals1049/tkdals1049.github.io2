<h2 data-ke-size="size26">1) 기본 문법 활용</h2>
<p data-ke-size="size18">- 데이터 타입의 길이</p>
<p data-ke-size="size16">- Java </p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li>short - 2byte</li>
<li>long - 8byte</li>
<li>char - 2byte</li>
</ul>
 
<p data-ke-size="size16">- Python </p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li>long - 무제한</li>
<li>float - 8byte</li>
</ul>
 
<p data-ke-size="size16">- C# </p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li>long - 8byte</li>
<li>char - 2byte</li>
<li><b>decimal</b><span> </span>- 16byte(실수형)</li>
</ul>
 
<p data-ke-size="size18">- 변수</p>
<p data-ke-size="size16">- 변수 명명 제한</p>
<p data-ke-size="size16">-&gt; 첫 자리에는 숫자를 사용할 수 없다.</p>
<p data-ke-size="size16">-&gt; 이미 사용되고 있는 예약어의 경우에는 변수로 사용할 수 없다.</p>
 
<p data-ke-size="size16">- 일반적인 변수 명명법</p>
<p data-ke-size="size16">프로그램의 가독성 향상 및 유지보수성 증가를 위해</p>
 
<p data-ke-size="size18">- 변수 명명법</p>
 
<p data-ke-size="size16"><b>Pascal Casing(파스칼 케이싱)</b><span> </span>: 대문자로 시작하는 변수명을 사용</p>
<p data-ke-size="size16">-&gt; 복잡어일 경우 중간에 시작하는 새로운 단어는 대문자로 적는다.</p>
<p data-ke-size="size16">-&gt;<span> </span><b>HumanResource</b></p>
 
<p data-ke-size="size16"><b>Camel Casing(카멜 케이싱)</b><span> </span>: 첫번째 문자는 소문자로 시작하고 복잡어일 경우 파스칼 케이싱과 동일</p>
<p data-ke-size="size16">-&gt;<span> </span><b>getName</b></p>
 
<p data-ke-size="size16"><b>GNU Naming(snake)</b><span> </span>: 모두 소문자를 사용하고 복합어 사이를 '_'를 사용하여 연결</p>
<p data-ke-size="size16">-&gt;<span> </span><b>human_resource</b></p>
 
<p data-ke-size="size16">-<span> </span><b>Hungarian notation(헝가리안 표기법)</b><span> </span>: 첫글자<span> </span><b>g는 전역변수</b>,<span> </span><b>m은 멤버변수</b><span> </span>의미</p>
<p data-ke-size="size16">-&gt; 전역이나 멤버변수의 경우 그 다음에 _를 적는다.</p>
<p data-ke-size="size16">-&gt;<span> </span><b>n과 i는 자연수</b>, i는 주로 인덱스, n은 카운트 목적에 주로 사용</p>
<p data-ke-size="size16">-&gt;<span> </span><b>g_name</b>,<span> </span><b>n_Cnt</b></p>
 
<p data-ke-size="size16">-<span> </span><b>GNU Naming Convention</b><span> </span>: 모든 문자를 대문자로 사용하는 GNU Naming Convention의 형태를 사용</p>
<p data-ke-size="size16">-&gt;<span> </span><b>DEFAULT_DEPARTMENT_CODE</b></p>
 
 
<p data-ke-size="size18">- 연산자</p>
<p data-ke-size="size16">산술 연산자, 시프트 연산자, 관계 연산자, 논리 연산자 등</p>
 
<p data-ke-size="size16">- 시프트 연산자</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li data-ke-size="size16">&gt;&gt; : 왼쪽 값에 오른쪽 값만큼의 부호비트를 채우면서 오른쪽으로 시프트 시킴</li>
<li data-ke-size="size16">&gt;&gt;&gt; : 왼쪽 값에 오른쪽 값만큼의 0으로 채우면서 오른쪽으로 시프트 시킴</li>
</ul>
 
<p data-ke-size="size16">- 비트 연산자</p>
<p data-ke-size="size16">&amp;(AND), |(OR), ^(XOR), ~(NOT)</p>
 
<p data-ke-size="size16">- 증감 연산자</p>
<p data-ke-size="size16">피연산자가 하나인 단항 연산자</p>
<p data-ke-size="size16">-&gt; 전위 증감 연산자와 후위 증감 연산자로 나뉨</p>
<p data-ke-size="size16">-&gt;<span> </span><b>대입 연산자와 함께 사용했을 때 동작의 순서가 다름</b></p>
 
<p data-ke-size="size16">- 삼항 연산자</p>
<p data-ke-size="size16">조건식의 결과에 따라 대입의 결과가 달라지는 경우 사용</p>
<p data-ke-size="size16">-&gt; String str = (a&gt;b) ? "학생 : "교수"</p>
 
<p data-ke-size="size18">- 사용자 정의 자료형</p>
<p data-ke-size="size16">기존에 존재하는 자료형에 새로운 이름을 부여하거나 여러 자료형을 묶어 새로운 자료형으로 사용하는 것</p>
<p data-ke-size="size16">-&gt; C언어에서는 typedef를 통한 별칭(alias)이나 구조체 의미</p>
<p data-ke-size="size16">-&gt;<span> </span><b>객체 지향 언어에서는 주로 클래스를 의미</b></p>
 
<p data-ke-size="size16">- C언어의 사용자 정의 자료형</p>
<pre id="code_1651778307250" class="angelscript" data-ke-language="c++" data-ke-type="codeblock"><code>typedef int INT;</code></pre>
<p data-ke-size="size16">-&gt; 새로운 이름 부여</p>
<pre id="code_1651778307251" class="cpp" data-ke-language="c++" data-ke-type="codeblock"><code>struct Human {
    char name[10];
    int age;
}</code></pre>
<p data-ke-size="size16">-&gt; 클래스와는 달리 함수(method)는 구조체에 포함될 수 없다.</p>
<p data-ke-size="size16">-&gt; 구조체에 함수<span> </span><b>포인터를 포함시키면 클래스와 유사한 동작 수행 가능</b></p>
<pre id="code_1651778307251" class="cpp" data-ke-language="c++" data-ke-type="codeblock"><code>typedef struct Human {
    char name[20];
    int age;
} Human;</code></pre>
 
<p data-ke-size="size16">- 객체지향에서의 사용자 정의 자료형 -&gt;<span> </span><b>클래스 이용</b></p>
 
<h3 data-ke-size="size23">2) 언어 특성 활용하기</h3>
<p data-ke-size="size16">- 프로그래밍 언어의 발전 과정</p>
<p data-ke-size="size16">세계 최초의 프로그램은 내장 방식의 프로그램</p>
 
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li>60년대 이전 :<span> </span><b>어셈블리</b>, 포트란,<span> </span><b>LISP</b></li>
<li>60년대 :<span> </span><b>COBOL</b>, PL/I, BASIC</li>
<li>70년대 :<span> </span><b>파스칼</b>,<span> </span><b>C</b>, SMALLTALK,<span> </span><b>PROLOG</b></li>
<li>80년대 : ADA,<span> </span><b>C++</b>,<span> </span><b>PERL</b>,<span> </span><b>파이썬</b></li>
<li>90년대 : 루비,<span> </span><b>자바</b>,<span> </span><b>자바스크립트</b>,<span> </span><b>PHP</b>, 비주얼 베이직</li>
<li>2000년대 이후 : C#, 스칼라, GO, KOTLIN 등</li>
</ul>
 
<p data-ke-size="size18">- 프로그래밍 언어의 유형</p>
 
<p data-ke-size="size16">- 개발 편의성에 따른 분류</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li><b>저급 언어</b><span> </span>: 기계어 수준,<span> </span><b>추상화 수준 낮음</b>, 프로그램 작성 어려움</li>
<li><b>고급 언어</b><span> </span>: 개발자가 쉽게 이해 가능,<span> </span><b>번역 과정 필요</b>, 컴파일러 또는 인터프리터 이용</li>
</ul>
 
<p data-ke-size="size16">- 구현 방식(패러다임)에 따른 분류</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li><b>절차 지향 언어</b></li>
<li><b>객체 지향 언어</b></li>
<li>함수형 언어 - LISP, SCALAR</li>
<li>논리형 언어 - PROLOG</li>
</ul>
 
<p data-ke-size="size16">- 실행방식에 따른 언어 유형</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li><b>컴파일 방식</b><span> </span>- 느린 번역,<span> </span><b>빠른 실행 속도</b></li>
<li><b>인터프리터 방식</b><span> </span>- 빠른 번역,<span> </span><b>느린 실행 속도</b></li>
<li><b>혼합형 - Java</b></li>
</ul>
 
<p data-ke-size="size18">-<span> </span><b>절차적 프로그래밍 언어</b></p>
<p data-ke-size="size16">절차 지향 혹은<span> </span><b>명령 지향적</b><span> </span>프로그래밍</p>
<p data-ke-size="size16">-&gt; 프로시저 호출의 개념을 바탕으로 하고 있는 프로그램</p>
<p data-ke-size="size16"> -&gt;<span> </span><b>프로시저, 루틴, 하위프로그램, 서브루틴, 메서드, 함수</b></p>
 
<p data-ke-size="size16">- 프로시저를 이용한 모듈성 장점</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li><b>코드의 재사용</b></li>
<li>프로그램 흐름 파악</li>
<li><b>모듈화 및 구조화</b></li>
</ul>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>매개변수와 반환 값</li>
<li><b>유효범위</b><span> </span>- 프로시저가 다른 프로시저의 변수에 접근하거나 그 반대의 경우를 막아줌</li>
<li>명시적 허가 - 명시적 허가 없이 방금전 자기 변수도 접근 불가</li>
<li><b>약결합</b><span> </span>-<span> </span><b>모듈성 강화</b></li>
</ul>
<p data-ke-size="size16">-<span> </span><b>객체지향</b><span> </span>프로그래밍 언어</p>
<p data-ke-size="size16">- 구조적(절차적) 방식의 문제점을 극복하고 인간이 사고하는 방식대로</p>
<p data-ke-size="size16">-<span> </span><b>객체(object)라는 실 세계에 존재하거나 또는 개념적인 그 어떤 것을 표현한 것</b></p>
<p data-ke-size="size16">- 기능과 자료가 통합된 객체 단위로 프로그래밍</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li><b>객체</b><span> </span>: 클래스의 인스턴스</li>
<li><b>클래스</b><span> </span>: 객체를 생성하는 틀</li>
<li><b>메세지</b><span> </span>:<span> </span><b>객체간 통신</b>을 의미</li>
</ul>
 
<p data-ke-size="size16">- 객체의 구성요소</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li>개체(Entity) : 현실 세계에 보이는 본질</li>
<li><b>속성(Attribute)</b><span> </span>: 자료 저장소 역할을 하며, 절차 지향 프로그래밍<span> </span><b>변수와 대응됨</b></li>
<li><b>메소드(Method)</b><span> </span>: 호출 단위 의미, 절차 지향 프로그램의<span> </span><b>함수와 대응됨</b></li>
</ul>
 
<p data-ke-size="size16">- 대표적 객체지향 언어</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li><b>Java</b><span> </span>: (과거)썬 마이크로시스템즈에서 개발</li>
<li><b>C#</b><span> </span>: 마이크로소프트에서 개발, 닷넷 프레임워크의 한 부분</li>
<li>델파이</li>
<li><b>스위프트</b><span> </span>: Objective-C와 공존할 목적</li>
</ul>
 
<p data-ke-size="size18">- 객체지향 프로그램의 특징</p>
<p data-ke-size="size16">-&gt;<span> </span><b>캡슐화, 추상화, 다형성, 정보은닉, 상속</b></p>
 
<p data-ke-size="size16">- 캡슐화와 정보은닉</p>
<p data-ke-size="size16"><b>캡슐화</b>는<span> </span><b>기능과 데이터를 하나로 묶어 객체로 표현</b>하는 것</p>
<p data-ke-size="size16">-&gt; 클래스<span> </span><b>내부의 동작으로 모르더라도 클래스가 제공하는 메소드를 사용해 클래스의 기능 사용</b></p>
<p data-ke-size="size16">-&gt; 캡슐화를 통해<span> </span><b>정보은닉이 가능해짐</b></p>
 
<p data-ke-size="size16">-<span> </span><b>정보은닉</b>의 범위</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li><b>private</b><span> </span>: 동일 클래스에서만 접근 가능</li>
<li>package : 동일 클래스, 동일 패키지</li>
<li><b>protected</b><span> </span>: 동일 클래스, 동일 패키지, 상속 관계</li>
<li><b>public</b><span> </span>: 모든 위치에서 젖ㅂ근 가능</li>
</ul>
 
 
 
 
 
<p data-ke-size="size16">-<span> </span><b>추상화</b></p>
<p data-ke-size="size16">실체에서<span> </span><b>공통되는 속성이나 관심있는 부분만 추출하여 모델링한 개념</b></p>
 
<p data-ke-size="size16">-<span> </span><b>상속</b> </p>
<p data-ke-size="size16">상위 클래스의 속성,<span> </span><b>메소드를 상속하여 새로운 클래스를 생성</b>하는 기법</p>
<p data-ke-size="size16">-&gt; 추<b>상화된 클래스를 구체화시키는 방법</b></p>
<p data-ke-size="size16">-&gt; C++은 다중상속 지원, Java에서는 클래스의 다중상속은 지원하지 않지만<span> </span><b>Interface의 다중구현 가능</b></p>
 
<p data-ke-size="size16">-<span> </span><b>다형성</b></p>
<p data-ke-size="size16">동일한 이름의 오퍼레이션(메소드)이 각 클래스마다 다른 사양으로 정의될 수 있는 것</p>
<p data-ke-size="size16">-&gt; 하나의 클래스 안에서 사용되는<span> </span><b>오버로딩과</b><span> </span>상속관계의 클래스에서 사용되는<span> </span><b>오버라이딩이</b><span> </span>대표적</p>
 
<p data-ke-size="size16">*<span> </span><b>오버로딩</b><span> </span>: 메소드 중복정의 -&gt; 매개변수 타입에 따른 구분</p>
<p data-ke-size="size16">*<span> </span><b>오버라이딩</b><span> </span>: 메소드 재정의 -&gt; 클래스에 따른 구분</p>
 
<p data-ke-size="size16">-&gt; 오버라이딩은 객체지향 설계의 OCP(Open Close Principle), LSP(Liskov Substitution Principle), DIP(Dependency Inversion Principle)를 구현하기 위한 필수 요소</p>
<p data-ke-size="size16"> -&gt;<span> </span><b>다형성의</b><span> </span>핵심, 다양한<span> </span><b>디자인 패턴에서 사용됨</b></p>
 
<p data-ke-size="size18">-<span> </span><b>스크립트 언어</b></p>
<p data-ke-size="size16"><b>소스코드를 컴파일하지 않고도 실행 할 수 있는 프로그래밍 언어</b></p>
<p data-ke-size="size16">-&gt; 주로 다른 응용 프로그램에 삽입되어 인터프리터를 통해 해석되고 동작</p>
<p data-ke-size="size16">  -&gt; 인터프리터가 있다면 어떤 운영 체제 위에서도 동일하게 동작</p>
 
<p data-ke-size="size16">- 대표적인 스크립트 언어</p>
<ul style="list-style-type: disc;" data-ke-list-type="circle">
<li><b>JavaScript</b><span> </span>: 미국 넷스케이프에서 개발 -&gt; Java와 기능상 아무런 관련이 없다.</li>
<li><b>JSP</b><span> </span>: Java Server Pages의 약자, 웹 페이지 동적 생성</li>

<h3 id="toc34" data-ke-size="size23"><span><b>객체지향 프로그래밍 언어의 구성요소</b></span></h3>
<p data-ke-size="size16"><span>🧙&zwj;♂️ 객체지향 프로그래밍 언어의 구성 요소에는<span> </span><b>객체(Object), 클래스(Class), 메시지(Message)</b>가 있음</span></p>
 
<h4 id="toc351" data-ke-size="size20"><span><b>객체(Object)</b></span></h4>
<p data-ke-size="size16"><span>✔ 데이터(속성)와 이를 처리하기 위한 연산(메소드)을 결합시킨 실체</span></p>
<p data-ke-size="size16"><span>✔ 데이터 구조와 그 위에서 수행되는 연산들을 가지고 있는 소프트웨어 모듈</span></p>
 
<p data-ke-size="size16"><span>➡속성(Attribute):</span></p>
<p data-ke-size="size16"><span>한 클래스 내에서 속한 객체들이 가지고 있는 데이터 값들을 단위별로 정의하는 것으로서</span></p>
<p data-ke-size="size16"><span>성질, 분류, 식별, 수량 또는 현재 상태 등을 표현</span></p>
 
<p data-ke-size="size16"><span>➡ 메소드(Method):</span></p>
<p data-ke-size="size16"><span>객체가 메시지를 받아 실행해야 할 때 구체적인 연산을 정의하는 것으로,</span></p>
<p data-ke-size="size16"><span>객체의 상태를 참조하거나 변경하는 수단이 됨</span></p>
 
 
 
<h4 id="toc352" data-ke-size="size20"><span><b>클래스(Class)</b></span></h4>
<p data-ke-size="size16"><span>✔ 두 개 이상의 유사한 객체들을 묶어서 하나의 공통된 특성을 표현하는 요소</span></p>
<p data-ke-size="size16"><span>✔ 객체의 유형 또는 타입(Object Type)을 의미</span></p>
 
<p data-ke-size="size18"> </p>
<h4 id="toc353" data-ke-size="size20"><span><b>메시지(Message)</b></span></h4>
<p data-ke-size="size16"><span>✔ 객체들 간 상호작용에 사용되는 수단, 메소드(동작/연산)를 일으키는 외부의 요구사항</span></p>
<p data-ke-size="size16"><span>✔<span style="color: #8a3db6;" data-darkreader-inline-color=""><u><span> </span><b>메시지를 받은 객체는 대응하는 연산을 수행하여 예상된 결과를 반환</b></u></span></span></p>
 
 
 
<h3 id="toc35" data-ke-size="size23"><span><b>객체지향 프로그래밍 언어의 특징</b></span></h3>
<p data-ke-size="size16"><span><b>🧙&zwj;♂️ 캡슐화, 정보 은닉, 추상화, 상속성, 다형성</b></span></p>
 
<h4 id="toc361" data-ke-size="size20"><span><b>캡슐화(Encapsulation)</b></span></h4>
<p data-ke-size="size16"><span><b>✔ 데이터(속성)과 데이터를 처리하는 함수를 하나로 묶는 것</b></span></p>
<p data-ke-size="size16"><span><b>✔ 캡슐화된 객체의 세부 내용이 외부에 은폐엄폐(정보 은닉)되어 변경 발생시 오류 파급효과 적음</b></span></p>
<p data-ke-size="size16"><span><b>✔ 재사용 용이</b></span></p>
 
 
<h4 id="toc362" data-ke-size="size20"><span><b>정보 은닉(Information Hiding)</b></span></h4>
<p data-ke-size="size16"><span><b>✔ 감기약을 예로 들면 정보 은닉은 감기약에 어떤 재료가 들어 있는지 몰라도</b></span></p>
<p data-ke-size="size16"><span><b>✔ 감기가 걸렸을 때 먹는 약이라는 것만 알고 복용하는 것과 같은 의미임</b></span></p>
 
 
 
<h4 id="toc363" data-ke-size="size20"><span><b>추상화(Abstraction)</b></span></h4>
<p data-ke-size="size16"><span><b>✔ 불필요한 부분은 생략하고 객체 속성 중 가장 중요한 것만 중점을 두어 개략화하는 것(모델화)</b></span></p>
<p data-ke-size="size16"><span><b>✔ 데이터의 공통된 성질을 추출하여 슈퍼 클래스를 선정하는 개념</b></span></p>
 
<p data-ke-size="size16"><span><b>➡ 과정 추상화: 자세한 수행과정 정의하지 않고 전반적인 흐름만 파악할 수 있게 설계</b></span></p>
<p data-ke-size="size16"><span><b>➡ 데이터 추상화: 데이터 세부속성이나 용도를 정의하지 않고 데이터 구조를 대표할 수 있는 표현으로 대체</b></span></p>
<p data-ke-size="size16"><span><b>➡ 제어 추상화: 이벤트 발생의 정확한 절차나 방법을 정의하지 않고, 대표할 수 있는 표현으로 대체</b></span></p>
 
 
<h4 id="toc364" data-ke-size="size20"><span><b>상속성(Inheritance)</b></span></h4>
<p data-ke-size="size16"><span><b>✔ 이미 정의된 상위 클래스(부모 클래스)의 모든 속성을 하위 클래스가 물려받는 것</b></span></p>
<p data-ke-size="size16"><span><b>✔ 하위 클래스는 상위 클래스의 모든 속성과 연산을 자신의 클래스 내에서 다시 정의하지 않고서도 즉시 자신의 속성으로 사용 가능</b></span></p>
 
<p data-ke-size="size16"><span><b>➡ 단일 상속: 하나의 상위 클래스로부터 상속받는 것</b></span></p>
<p data-ke-size="size16"><span><b>➡ 다중 상속: 여러 개의 상위클래스로부터 상속받는 것</b></span></p>
 
 
<h4 id="toc365" data-ke-size="size20"><span><b>다형성(Polymorphism)</b></span></h4>
<p data-ke-size="size16"><span><b>✔ 하나의 메시지에 대해 각 객체(클래스)가 가지고 있는 고유한 방법(특성)으로 응답할 수 있는 능력을 의미함</b></span></p>
<p data-ke-size="size16"><span><b>✔ 객체(클래스)들은 동일한 메소드명을 사용하며 같은 의미의 응답을 함</b></span></p>
 
 
 
<h3 id="toc36" data-ke-size="size23"><span><b>객체지향 기술(복습)</b></span></h3>
<p data-ke-size="size16"><span><b>✔ 객체지향 분석</b></span></p>
<p data-ke-size="size16"><span><b>(1)<span> </span><span style="color: #ee2323;">객</span>체 모델링: 메소드 식별</b></span></p>
<p data-ke-size="size16"><span><b>(2)<span> </span><span style="color: #ee2323;">동</span>적 모델링: 기능의 흐름 표시</b></span></p>
<p data-ke-size="size16"><span><b>(3)<span> </span><span style="color: #ee2323;">기</span>능 모델링: 상세분석(제한사항, 성능 등)</b></span></p>
 
<p data-ke-size="size16"><b><span>✔ 객체지향 분석 방법론</span></b></p>
<p data-ke-size="size16"><b><span>(1) 럼바우(Rumbaugh): 객체, 동적, 기능 모형으로 분리하여 접근</span></b></p>
<p data-ke-size="size16"><b><span>(2) 부치(Booch): 미시적(micro), 거시적(macro) 개발 프로세스로 접근</span></b></p>
<p data-ke-size="size16"><b><span>(3) Coad&amp;Yourdon: E-R 다이어그램 사용하여 모델링</span></b></p>
<p data-ke-size="size16"><b><span>(4) Jacobson: 시나리오 접근</span></b></p>
<p data-ke-size="size16"><b><span>(5) Wirfs-Brocks: 설계의 연속적인 프로세스</span></b></p>
 
 
<h3 id="toc37" data-ke-size="size23"><span><b>스크립트 언어</b></span></h3>
<p data-ke-size="size16"><span><b>🧙&zwj;♂️ HTML 문서 안에 직접 프로그래밍 언어를 삽입하여 사용하는 것</b></span></p>
<p data-ke-size="size16"><span><b>🧙&zwj;♂️ 기계어로 컴파일 되지 않고 별도의 번역기가 소스를 분석하여 동작하게 함</b></span></p>
 
<p data-ke-size="size16"><span><b>✔ 게시판 입력, 상품 검색, 회원 가입 등과 같은 DB 처리 작업에 주로 사용</b></span></p>
 
<p data-ke-size="size16"><span style="color: #009a87;" data-darkreader-inline-color=""><b>▶ 서버에서 해석되어 실행된 후 결과만 클라이언트로 보냄 = 서버용 스크립트 언어</b></span></p>
<p data-ke-size="size16"><span style="color: #009a87;" data-darkreader-inline-color=""><b><b>➡서버용 스크립트 언어: ASP, JSP, PHP, 파이썬</b></b></span></p>
 
<p data-ke-size="size16"><span style="color: #8a3db6;" data-darkreader-inline-color=""><b>▶ 클라이언트의 웹브라우저에서 해석+실행 = 클라이언트용 스크립트 언어</b></span></p>
<p data-ke-size="size16"><span style="color: #8a3db6;" data-darkreader-inline-color=""><b><b>➡ 클라이언트용 스크립트 언어: 자바스크립트(JAVA Script), VB스크립트(Visual Basic Script)</b></b></span></p>

<p data-ke-size="size16"><span><b>🧙&zwj;♂️ 프로그램이 수행해야 하는 문제를 기술하는 언어</b></span></p>
<p data-ke-size="size16"><span><b>🧙&zwj;♂️ 알고리즘은 명시하지 않고 목표를 명시함</b></span></p>
<p data-ke-size="size16"><span><b>🧙&zwj;♂️ 함수형 언어, 논리형 언어 등이 있음</b></span></p>
<p data-ke-size="size16"><span style="color: #ee2323;" data-darkreader-inline-color=""><b><b><b>🧙&zwj;♂️ 명령형 언어의 반대개념임</b></b></b></span></p>
 
<p data-ke-size="size16"><span style="color: #006dd7;" data-darkreader-inline-color=""><b><b><b>▶ 가독성이나 재사용성이 좋음</b></b></b></span></p>
<p data-ke-size="size16"><span style="color: #006dd7;" data-darkreader-inline-color=""><b><b><b>▶ 작동 순서가 구체적으로 작성되지 않기 때문에 오류가 적음</b></b></b></span></p>
<p data-ke-size="size16"><span style="color: #006dd7;" data-darkreader-inline-color=""><b><b><b>▶ 프로그램 동작을 변경하지 않고도 관련 값을 대체할 수 있음</b></b></b></span></p>
 
<p data-ke-size="size16"><span><b>✔ HTML, LISP, PROLOG, XML, Haskell 등</b></span></p>
<p data-ke-size="size16"><br /><b><span>프로그래밍 언어의 언어별 특성</span></b></p>

<h2 id="신기술동향" data-ke-size="size26">신기술동향</h2>
<h4 id="재해복구-시스템" data-ke-size="size20">재해복구 시스템</h4>
<p data-ke-size="size16">복구목표시간 (RTO, Recovery Time Objective)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>재해로 인하여 서비스가 중단되었을 때, 서비스를 복구하는데까지 걸리는 최대 허용시간</li>
</ul>
<p data-ke-size="size16">복구목표시점 (RPO, Recovery Point Objective)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>재해로 인하여 중단된 서비스를 복구하였을 때, 유실을 감내할 수 있는 데이터의 손실 허용시점</li>
</ul>
<p data-ke-size="size16">업무연속성계획 (BCP, Business Continuity Planning)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>정보기술부문뿐 아니라, 인력-설비-자금 등 제반 자원을 대상으로 장애 및 재해를 포괄하여 조직의 생존을 보장하기 위한 예방 및 복구활동 등을 포함하는 보다 광범위한 계획</li>
</ul>
<p data-ke-size="size16">재해복구계획 (DRP, Disaster Recovery Planning)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>정보기술서비스기반에 대하여 재해가 발생하는 경우를 대비하여, 이의 빠른 복구를 통해 업무에 대한 영향을 최소화하기 위한 제반 계획</li>
</ul>
<p data-ke-size="size16">재해복구시스템 (DRS, Disaster Recovery System)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>재해복구계획의 원활한 수행을 지원하기 위하여 평상시에 확보하여 두는 인적-물적 자원 및 이들에 대한 지속적인 관리체계가 통합된 것</li>
</ul>
<p data-ke-size="size16">주센터</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>현재 사용중인 전산 인프라를 운영하는 전산센터로써, 주전산센터 혹은 주사이트라 일컫기도 함.</li>
</ul>
<p data-ke-size="size16">재해복구 (DR, Disaster Recovery)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>재해로 인하여 중단된 정보기술 서비스를 재개하는 것</li>
</ul>
<p data-ke-size="size16">재해복구센터</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>주 센터에 반하여 재해에 대비하여 업무 연속성을 보장할 수 있도록 원격지에 구축한 전산 센터로써, 원격지센터 혹은 백업 센터라 일컫기도 함</li>
</ul>
<h4 id="재해복구시스템의-복구수준별-유형-비교" data-ke-size="size20">재해복구시스템의 복구수준별 유형 비교</h4>
<p data-ke-size="size16">Mirror Site</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>주 센터와 동일한 수준의 정보기술 자원을 원격지에 구축, Active-Active 상태로 실시간 동시 서비스 제공</li>
</ul>
<p data-ke-size="size16">Hot Site</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>주 센터와 동일한 수준의 정보기술 자원을 원격지에 구축하여 Standby 상태로 유지 (Active-Standby)</li>
</ul>
<p data-ke-size="size16">Warm Site</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>중요성이 높은 정보기술자원만 부분적으로 재해복구센터에 보유</li>
<li>데이터는 주기적(약 수시간~1일)으로 백업</li>
</ul>
<p data-ke-size="size16">Cold Site</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>데이터만 원격지에 보관하고, 이의 서비스를 위한 정보자원은 확보하지 않거나 장소 등 최소한으로만 확보</li>
<li>주센터의 데이터는 주기적(수일~수주)으로 원격지에 백업</li>
</ul>
<h4 id="클라우드-컴퓨팅" data-ke-size="size20">클라우드 컴퓨팅</h4>

<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>인프라(infra)
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>정보기술이나 인터넷에서 컴퓨터와 사용자를 연결하는데 사용되는 물리적 하드웨어를 지칭. IT서비스를 제공하기 위한 서버와 스토리지 같은 H/W와 각 장비들을 연결하고 외부 인터넷망과 연결하기 위한 네트워크(장비 및 S/W), 나아가 이런 장비들을 유지관리하기 위해 필요한 데이터센터(랙, 공조, 항온항습, 전기, 변압, UPS, 소화설비를 포함하는)까지도 의미한다.</li>
</ul>
</li>
</ul>

</li>

<h4 id="가상화virtualization" data-ke-size="size20">가상화(Virtualization)</h4>
<ul style="list-style-type: disc;" data-ke-list-type="disc">

<h4 id="quartz-cron-trigger" data-ke-size="size20">Quartz Cron Trigger</h4>
<pre class="excel"><code>초(Seconds)
분(Minutes)
시(Hours)
일(Day-of-Month)
월(Months)
요일(Days-of-Week)
연도(Year) - optional
</code></pre>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>와일드카드 문자
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>이 필드에서 가능한 &ldquo;매번&rdquo;을 표현합니다. 그러므로, 앞의 예제에서 &ldquo;월(Month)&rdquo;필드의 &lsquo;&rsquo;표시는 단순히 &ldquo;매월&rdquo;을 의미합니다. 요일(Day-of-Week)필드에 &lsquo;&rsquo;는 &ldquo;한주동안 매일&rdquo;을 의미합니다.</li>
</ul>
</li>
<li>월(Months)
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>숫자 0에서 11, 또는 물자열 JAN, FEB, MAR, APR, MAY, JUN. JUL, AUG, SEP, OCT, NOV, 그리고 DEC를 사용할 수 있습니다. 요일(Days-of-Week)은 1과 7(1=일요일)사이의 숫자를 사용하거나, 문자열 SUN, MON, TUE, WED, THU, FRI, 그리고, SAT를 사용할 수 있습니다.</li>
</ul>
</li>
<li>&ldquo;/&rdquo; 문자
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>값의 증가 표현에 사용될 수 있습니다. 예를들어, 분(Minutes) 필드에 &lsquo;0/15&rsquo; 를 사용한다면, 이것은 &ldquo;0분 부터 시작하여 (시간의) 매 15분 마다&rdquo;를 의미합니다. 만약, 분(Minutes) 필드에 &lsquo;3/20&rsquo;을 사용한다면, &ldquo;3분부터 시작하여 매 20분 마다&rdquo;를 의미하므로, &lsquo;3,23,43&rsquo;을 분 필드에 나열한것과 같습니다. 미묘하게 주목할 점은 &ldquo;/35&rdquo;의 경우 &ldquo;매 35분 마다&rdquo;를 의미하지 않는다는 것입니다. 이것은, &ldquo;0분부터 시작하여 매 35분 마다&rdquo;이므로, 달리 표현하면 &lsquo;0,35&rsquo;를 나열한 것과 같습니다.</li>
</ul>
</li>
<li>&ldquo;?&rdquo;문자
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>일(day-of-month)와 요일(day-of-week) 필드에서만 허용됩니다. 이것은 &ldquo;설정 값 없음&rdquo;에 사용됩니다. &ldquo;?&rdquo;문자는 두 필드(일, 요일)중 하나만 설정할 경우에 나머지 필드에 유용하게 사용됩니다. 다음 섹션의 예제를 보면 이해가 쉬울 것입니다.</li>
</ul>
</li>
<li>&ldquo;L&rdquo;문자
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>일(day-of-month)과 요일(day-of-week) 필드에서만 허용됩니다. 이 문자는 &ldquo;마지막(last)&rdquo;의 첫글자에서 따왔습니다. 하지만, 두 필드에서 조금 다른 의미로 해석됩니다. 예를들어, &ldquo;L&rdquo;이 일(day-of-month) 필드에서 사용되었을 경우 &ldquo;이달의 마지막 날&rdquo;을 의미합니다. 1월이면 31일이 되고, 2월은 (윤년이 아닌경우)28일이 됩니다. 요일(day-of-week) 필드에서 사용될 경우에는 단순히 &ldquo;7&rdquo; 또는 &ldquo;SAT&rdquo;를 의미합니다. 하지만, &ldquo;L&rdquo;이 특정값의 뒤에 올경우, 이것은 &ldquo;이달의 마지막 x요일&rdquo;이 됩니다. 예를들어, &ldquo;6L&rdquo; 또는 &ldquo;FRIL&rdquo;은 &ldquo;이달의 마지막 금요일&rdquo;을 의미합니다. 또한, 마지막 날에서 특정 기간(offset)을 설정할 수도 있습니다. &ldquo;L-3&rdquo;과 같이 설정하면 &ldquo;(이달의 마지막일로부터)3일전부터 마지막날 까지&rdquo;의 기간을 의미합니다. &ldquo;L&rdquo;옵션을 사용할때, 값의 목록이나 범위를 명확히 하지 않을 경우 예상밖의 결과를 낳을 수 있으므로 주의가 필요합니다.</li>
</ul>
</li>
<li>&ldquo;W&rdquo;
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>주어진 날로부터 가장 가까운 평일(월요일~금요일)을 가르킵니다. 만일, &ldquo;15W&rdquo;라는 값을 일(day-of-month)필드에 설정하면, &ldquo;이달의 15번째날에서 가장 가까운 평일&rdquo;을 의미합니다.</li>
</ul>
</li>
<li>&ldquo;#&rdquo;
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>&ldquo;이달의 n번째 X요일&rdquo;을 가르킵니다. 예를들어, &ldquo;6#3&rdquo; 또는 &ldquo;FRI#3&rdquo;을 요일(day-of-week) 필드에 설정할 경우 이것은 &ldquo;이달의 세번째 금요일&rdquo;을 의미 합니다.</li>
</ul>
</li>
</ul>
<h2 id="신기술동향" data-ke-size="size26">신기술동향</h2>
<h4 id="재해복구-시스템" data-ke-size="size20">재해복구 시스템</h4>
<p data-ke-size="size16">복구목표시간 (RTO, Recovery Time Objective)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>재해로 인하여 서비스가 중단되었을 때, 서비스를 복구하는데까지 걸리는 최대 허용시간</li>
</ul>
<p data-ke-size="size16">복구목표시점 (RPO, Recovery Point Objective)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>재해로 인하여 중단된 서비스를 복구하였을 때, 유실을 감내할 수 있는 데이터의 손실 허용시점</li>
</ul>
<p data-ke-size="size16">업무연속성계획 (BCP, Business Continuity Planning)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>정보기술부문뿐 아니라, 인력-설비-자금 등 제반 자원을 대상으로 장애 및 재해를 포괄하여 조직의 생존을 보장하기 위한 예방 및 복구활동 등을 포함하는 보다 광범위한 계획</li>
</ul>
<p data-ke-size="size16">재해복구계획 (DRP, Disaster Recovery Planning)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>정보기술서비스기반에 대하여 재해가 발생하는 경우를 대비하여, 이의 빠른 복구를 통해 업무에 대한 영향을 최소화하기 위한 제반 계획</li>
</ul>
<p data-ke-size="size16">재해복구시스템 (DRS, Disaster Recovery System)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>재해복구계획의 원활한 수행을 지원하기 위하여 평상시에 확보하여 두는 인적-물적 자원 및 이들에 대한 지속적인 관리체계가 통합된 것</li>
</ul>
<p data-ke-size="size16">주센터</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>현재 사용중인 전산 인프라를 운영하는 전산센터로써, 주전산센터 혹은 주사이트라 일컫기도 함.</li>
</ul>
<p data-ke-size="size16">재해복구 (DR, Disaster Recovery)</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>재해로 인하여 중단된 정보기술 서비스를 재개하는 것</li>
</ul>
<p data-ke-size="size16">재해복구센터</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>주 센터에 반하여 재해에 대비하여 업무 연속성을 보장할 수 있도록 원격지에 구축한 전산 센터로써, 원격지센터 혹은 백업 센터라 일컫기도 함</li>
</ul>
<h4 id="재해복구시스템의-복구수준별-유형-비교" data-ke-size="size20">재해복구시스템의 복구수준별 유형 비교</h4>
<p data-ke-size="size16">Mirror Site</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>주 센터와 동일한 수준의 정보기술 자원을 원격지에 구축, Active-Active 상태로 실시간 동시 서비스 제공</li>
</ul>
<p data-ke-size="size16">Hot Site</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>주 센터와 동일한 수준의 정보기술 자원을 원격지에 구축하여 Standby 상태로 유지 (Active-Standby)</li>

<p data-ke-size="size16">Warm Site</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>중요성이 높은 정보기술자원만 부분적으로 재해복구센터에 보유</li>
<li>데이터는 주기적(약 수시간~1일)으로 백업</li>
</ul>
<p data-ke-size="size16">Cold Site</p>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>데이터만 원격지에 보관하고, 이의 서비스를 위한 정보자원은 확보하지 않거나 장소 등 최소한으로만 확보</li>
<li>주센터의 데이터는 주기적(수일~수주)으로 원격지에 백업</li>
</ul>

<h4 id="quartz-cron-trigger" data-ke-size="size20">Quartz Cron Trigger</h4>
<pre class="excel"><code>초(Seconds)
분(Minutes)
시(Hours)
일(Day-of-Month)
월(Months)
요일(Days-of-Week)
연도(Year) - optional
</code></pre>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>와일드카드 문자
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>이 필드에서 가능한 &ldquo;매번&rdquo;을 표현합니다. 그러므로, 앞의 예제에서 &ldquo;월(Month)&rdquo;필드의 &lsquo;&rsquo;표시는 단순히 &ldquo;매월&rdquo;을 의미합니다. 요일(Day-of-Week)필드에 &lsquo;&rsquo;는 &ldquo;한주동안 매일&rdquo;을 의미합니다.</li>
</ul>
</li>
<li>월(Months)
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>숫자 0에서 11, 또는 물자열 JAN, FEB, MAR, APR, MAY, JUN. JUL, AUG, SEP, OCT, NOV, 그리고 DEC를 사용할 수 있습니다. 요일(Days-of-Week)은 1과 7(1=일요일)사이의 숫자를 사용하거나, 문자열 SUN, MON, TUE, WED, THU, FRI, 그리고, SAT를 사용할 수 있습니다.</li>
</ul>
</li>
<li>&ldquo;/&rdquo; 문자
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>값의 증가 표현에 사용될 수 있습니다. 예를들어, 분(Minutes) 필드에 &lsquo;0/15&rsquo; 를 사용한다면, 이것은 &ldquo;0분 부터 시작하여 (시간의) 매 15분 마다&rdquo;를 의미합니다. 만약, 분(Minutes) 필드에 &lsquo;3/20&rsquo;을 사용한다면, &ldquo;3분부터 시작하여 매 20분 마다&rdquo;를 의미하므로, &lsquo;3,23,43&rsquo;을 분 필드에 나열한것과 같습니다. 미묘하게 주목할 점은 &ldquo;/35&rdquo;의 경우 &ldquo;매 35분 마다&rdquo;를 의미하지 않는다는 것입니다. 이것은, &ldquo;0분부터 시작하여 매 35분 마다&rdquo;이므로, 달리 표현하면 &lsquo;0,35&rsquo;를 나열한 것과 같습니다.</li>
</ul>
</li>
<li>&ldquo;?&rdquo;문자
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>일(day-of-month)와 요일(day-of-week) 필드에서만 허용됩니다. 이것은 &ldquo;설정 값 없음&rdquo;에 사용됩니다. &ldquo;?&rdquo;문자는 두 필드(일, 요일)중 하나만 설정할 경우에 나머지 필드에 유용하게 사용됩니다. 다음 섹션의 예제를 보면 이해가 쉬울 것입니다.</li>
</ul>
</li>
<li>&ldquo;L&rdquo;문자
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>일(day-of-month)과 요일(day-of-week) 필드에서만 허용됩니다. 이 문자는 &ldquo;마지막(last)&rdquo;의 첫글자에서 따왔습니다. 하지만, 두 필드에서 조금 다른 의미로 해석됩니다. 예를들어, &ldquo;L&rdquo;이 일(day-of-month) 필드에서 사용되었을 경우 &ldquo;이달의 마지막 날&rdquo;을 의미합니다. 1월이면 31일이 되고, 2월은 (윤년이 아닌경우)28일이 됩니다. 요일(day-of-week) 필드에서 사용될 경우에는 단순히 &ldquo;7&rdquo; 또는 &ldquo;SAT&rdquo;를 의미합니다. 하지만, &ldquo;L&rdquo;이 특정값의 뒤에 올경우, 이것은 &ldquo;이달의 마지막 x요일&rdquo;이 됩니다. 예를들어, &ldquo;6L&rdquo; 또는 &ldquo;FRIL&rdquo;은 &ldquo;이달의 마지막 금요일&rdquo;을 의미합니다. 또한, 마지막 날에서 특정 기간(offset)을 설정할 수도 있습니다. &ldquo;L-3&rdquo;과 같이 설정하면 &ldquo;(이달의 마지막일로부터)3일전부터 마지막날 까지&rdquo;의 기간을 의미합니다. &ldquo;L&rdquo;옵션을 사용할때, 값의 목록이나 범위를 명확히 하지 않을 경우 예상밖의 결과를 낳을 수 있으므로 주의가 필요합니다.</li>
</ul>
</li>
<li>&ldquo;W&rdquo;
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>주어진 날로부터 가장 가까운 평일(월요일~금요일)을 가르킵니다. 만일, &ldquo;15W&rdquo;라는 값을 일(day-of-month)필드에 설정하면, &ldquo;이달의 15번째날에서 가장 가까운 평일&rdquo;을 의미합니다.</li>
</ul>
</li>
<li>&ldquo;#&rdquo;
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>&ldquo;이달의 n번째 X요일&rdquo;을 가르킵니다. 예를들어, &ldquo;6#3&rdquo; 또는 &ldquo;FRI#3&rdquo;을 요일(day-of-week) 필드에 설정할 경우 이것은 &ldquo;이달의 세번째 금요일&rdquo;을 의미 합니다.</li>
</ul>
</li>
</ul>
